# Reflection

## Overview

AIStack Audit was built as a SaaS-style MVP focused on solving AI subscription overspending and optimization problems.

The project combined:
- frontend engineering
- backend API development
- product thinking
- SaaS workflows
- user research
- deployment
- documentation

---

# What I Learned

## Full-Stack Development

This project helped me improve my understanding of:
- React component architecture
- State management
- Dynamic forms
- Express backend APIs
- Frontend-backend communication
- Public route handling
- Deployment workflows

---

## Product Thinking

One of the biggest learnings was understanding that real products involve much more than coding.

The project required:
- User validation
- Market thinking
- Feature prioritization
- SaaS workflow planning
- Shareability and growth considerations

---

## User Research Insights

User interviews revealed several common problems:
- Multiple overlapping AI subscriptions
- Lack of centralized AI spending visibility
- Inactive or underused subscriptions
- Difficulty comparing AI tool value

These insights validated the core idea behind AIStack Audit.

---

# Biggest Challenges

## Dynamic Audit Logic

Designing recommendation logic that remained explainable and predictable was challenging.

Instead of relying entirely on AI-generated recommendations, the project used deterministic rule-based audit logic.

---

## Shareable Public Reports

Implementing public audit URLs required:
- React Router integration
- Backend audit retrieval APIs
- Dynamic route rendering
- UUID-based report handling

---

## Deployment

Deploying frontend and backend separately introduced:
- Production API URL management
- Environment differences
- Deployment debugging
- Routing challenges

---
